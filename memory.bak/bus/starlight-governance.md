# 群星体系 · 三层治理架构 (Starlight Governance)

> 版本：v2.0 | 日期：2026-06-22
> 核心哲学：聚是一团火，散是满天星
> 设计理念：三层互补治理，集权统御，分权赋能，涌现智慧

## 一、三层架构总览

```
                    ╔═══════════════╗
                    ║    集 权 层    ║ ← 顶层：全局战略
                    ║  (麒麟主控)    ║
                    ║  聚是一团火    ║
                    ╚═══════╤═══════╝
                            │ 统御
                    ╔═══════▼═══════╗
                    ║    群 星 层    ║ ← 中层：缓冲分权
                    ║  (Agent 协作)  ║
                    ║  各司其职      ║
                    ╚═══════╤═══════╝
                            │ 赋能
                    ╔═══════▼═══════╗
                    ║   蜂 群 层    ║ ← 底层：即时涌现
                    ║  (Agent 碰撞)  ║
                    ║  散是满天星    ║
                    ╚═══════════════╝
```

## 二、集权层（顶层）

### 核心价值
全局战略、统一调度、秩序底线

### 解决的问题
- 群星无统筹 → 集权提供全局视角
- 蜂群无长远目标 → 集权设定方向和边界

### 执行主体
麒麟 (main) — 唯一决策者

### 职责范围
1. **战略决策** — 确定公司方向、优先级、资源分配
2. **统一调度** — 决定哪些任务走哪一层
3. **秩序底线** — 制定红线规则，玄武监督执行
4. **最终裁决** — 蜂群共识与集权判断冲突时，集权说了算
5. **知识沉淀** — 吸收各层输出的精华

### 触发条件
- 简单/紧急任务：麒麟直接执行，不经中层
- 战略级决策：麒麟直接拍板
- 跨层冲突：麒麟仲裁
- 任务路由：决定任务走哪一层处理

### 边界
- 不陷入细节执行（那是中层和底层的活）
- 不频繁干预蜂群对话（除非触发安全红线）
- 不做微观管理（相信中层和底层的自主性）

## 三、群星层（中层）

### 核心价值
缓冲分权、独立生存、差异化发展

### 解决的问题
- 集权僵化 → 群星提供灵活性
- 约束底层蜂群无序扩张 → 群星划定协作边界

### 执行主体
16 虾兵 — 各自为战又相互呼应

### 职责范围
1. **任务执行** — 接收麒麟分派的独立任务，自主完成
2. **领域深耕** — 每个虾兵在自己领域做到极致
3. **横向协作** — 虾兵之间可以协商合作（经麒麟授权）
4. **异常上报** — 遇到超出能力范围的任务，及时上报麒麟
5. **自我进化** — 在实践中积累经验，提升 skill_level

### 触发条件
- 大多数常规任务：走群星层，由最匹配的虾兵执行
- 多步骤任务：麒麟拆解后分派给多个虾兵并行
- 虾兵之间需要协作：麒麟授权后直接对接

### 边界
- 不能擅自改变公司战略（那是集权层的活）
- 不能越级向蜂群层发布议题（需经集权层批准）
- 不能自行扩展权限范围（需麒麟授权）
- 不能绕过玄武安全审查

## 四、蜂群层（底层）

### 核心价值
即时自适应、高效并行执行

### 解决的问题
- 集权审批滞后 → 蜂群即时响应
- 群星单元一线调度繁琐 → 蜂群自主碰撞

### 执行主体
任意数量的虾兵 — 按需组合，自由碰撞

### 职责范围
1. **创意碰撞** — 多 Agent 自由讨论，涌现创新方案
2. **快速原型** — 短时间内生成多个备选方案
3. **压力测试** — 从多角度质疑和验证一个方案
4. **市场模拟** — 模拟不同场景下的 Agent 行为
5. **共识达成** — 通过质量加权投票选出最优方案

### 触发条件
- 创意/策略类任务：麒麟授权后启动蜂群
- 多方案对比：用户明确要求时自动启用
- 风险评估：需要多角度验证时
- 突发事件：集权层来不及反应时的快速响应

### 边界
- 没有最终决策权（共识方案需经集权层确认）
- 不能脱离玄武安全护栏（实时监听）
- 不能无限膨胀（规模由集权层控制）
- 不能持久存在（蜂群结束后自动解散）

## 五、三层协同机制

### 5.1 任务路由规则

```
收到任务
  ↓
集权层评估：
  ├─ 简单/紧急 → 集权层直接执行
  ├─ 常规/独立 → 群星层分派给对应虾兵
  └─ 创意/复杂 → 蜂群层碰撞 → 群星层执行 → 集权层确认
```

### 5.2 信息流向

```
自上而下：
集权层 → 战略目标、优先级、红线规则
群星层 → 领域知识、执行反馈
蜂群层 → 涌现方案、创新思路

自下而上：
蜂群层 → 共识方案、多维验证
群星层 → 执行结果、领域洞察
集权层 → 最终决策、战略调整
```

### 5.3 冲突解决

| 冲突类型 | 解决方式 |
|---------|---------|
| 蜂群共识 ≠ 集权判断 | 集权层最终裁决 |
| 群星层之间资源竞争 | 集权层按优先级分配 |
| 群星层越权 | 玄武拦截 + 集权层处罚 |
| 蜂群层失控 | 玄武终止 + 集权层复盘 |

### 5.4 进化机制

每层都有自我进化的能力：

- **集权层**：通过吸收各层反馈，优化调度策略
- **群星层**：每个虾兵通过实践提升 skill_level
- **蜂群层**：通过知识沉淀，涌现模式逐渐固化为可复用规则

## 六、聚散之道

### 聚（集权层主导）
- 战略目标统一
- 资源集中调配
- 秩序底线一致
- 关键时刻集中决策

### 散（蜂群层主导）
- 虾兵独立执行
- 蜂群自由碰撞
- 领域差异化发展
- 即时自适应响应

### 聚散转换
- 常态：散（群星层独立执行，蜂群层按需涌现）
- 危机：聚（集权层收回控制权，统一调度）
- 创新：先散后聚（蜂群碰撞 → 共识 → 集权确认 → 执行）
- 稳定：先聚后散（集权定方向 → 群星分解 → 各自执行）

## 七、核心格言

> **聚是一团火** — 集权层统御全局，凝聚力量
> **散是满天星** — 蜂群层自由涌现，各自发光
> **群星居中** — 缓冲分权，承上启下
