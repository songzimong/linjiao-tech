# 麒麟体系 · 任务编排器 (Task Orchestrator)

> 版本：v1.0 | 日期：2026-06-22

## 任务状态机

```
PENDING → DISPATCHED → RUNNING → REVIEWING → DONE
                              ↓
                          FAILED → RETRY(≤3) → FAILED → ARCHIVED
```

## 任务模板

### 模板 1: 简单任务 (单 Agent)
```
用户请求 → 麒麟解析 → 匹配 Agent → spawn → 等待结果 → 验收 → 回复
```
适用：独立任务，无需协作

### 模板 2: 并行任务 (多 Agent)
```
用户请求 → 麒麟拆解 → [spawn A, spawn B, spawn C] → yield 等待 → 汇总结果 → 验收 → 回复
```
适用：多个独立子任务

### 模板 3: 流水线任务 (有依赖)
```
用户请求 → 麒麟拆解 → spawn A → 等 A 完成 → 提取 A 输出 → spawn B → 等 B 完成 → 汇总 → 验收 → 回复
```
适用：B 需要 A 的输出

### 模板 4: 跨域任务 (多殿协作)
```
用户请求 → 麒麟解析 → 查白泽(历史上下文) → 匹配能力 → spawn → 结果写回对应殿堂 → 白泽更新索引 → 验收 → 回复
```
适用：需要记忆写入的任务

## DAG 编排示例

对于复杂任务，麒麟自动构建有向无环图：

```
任务: "调研某个技术趋势并写报告"

                    [白泽:查历史]
                         ↓
                    [青龙:技术调研] ──┐
                         ↓            │
                    [朱雀:内容整理] ──┤
                         ↓            │
                    [玄武:安全审查] ──┘
                         ↓
                    [麒麟:汇总报告]
```

并行分支用 `sessions_spawn` + `sessions_yield` 实现。

## 错误恢复

1. Agent 超时 (5 分钟): 重试一次
2. 重试失败: 切换到备选 Agent
3. 备选也失败: 麒麟降级处理或告知用户
4. 记录失败到 `bus/audit-log.json`

## 性能指标

- 简单任务: < 2 分钟
- 并行任务: max(子任务耗时) + 汇总时间
- 流水线任务: sum(子任务耗时)
- 超时阈值: 10 分钟 (麒麟主动中断)
