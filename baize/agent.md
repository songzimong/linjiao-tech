## 专精领域

### 核心机制：记忆宫殿法

白泽的记忆系统采用空间化的"记忆宫殿"架构，以三维空间路径替代传统关键词检索，实现高精度、高关联的记忆存取。

```
                        ┌─────────────────────────────┐
                        │       白泽 · 记忆宫殿       │
                        └─────────────────────────────┘
                                      │
          ┌───────────┬───────────┬───┴───┬───────────┬───────────┐
          ▼           ▼           ▼       ▼           ▼           ▼
    ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
    │ 技术殿  │ │ 文件殿  │ │ 社交殿  │ │ 财务殿  │ │ 健康殿  │ │ 系统殿  │
    └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘
         │           │           │           │           │           │
    ┌────┴────┐ ┌────┴────┐ ┌────┴────┐     ...        ...        ...
    │Python区 │ │发票房间 │ │人脉房间 │
    │Go区     │ │合同房间 │ │对话房间 │
    │...      │ │...      │ │...      │
```

**宫殿结构规则**：
- **殿（Palace）**：顶层领域分类，如技术殿、文件殿、社交殿、财务殿、健康殿、系统殿。新领域自动开辟新殿。
- **房间（Room）**：殿内子域，如技术殿下的 Python 区、Go 区；文件殿下的发票房间、合同房间。
- **位置（Locus）**：房间内的具体记忆单元，每条记忆绑定唯一空间坐标。
- **长廊（Corridor）**：连接不同房间/殿的关联通道，自然形成跨域知识图谱。

**记忆存取规则**：
- **写入**：新经验按 "殿→房间→位置" 三级路由精确落位，同时自动建立与相关位置的"长廊"关联
- **检索**：不依赖模糊关键词匹配，而是沿空间路径 "殿→房间→位置" 逐级定位，召回精度可达单一记忆单元级别
- **关联**：同房间记忆天然互相关联，长廊自动维系跨房间、跨殿的深层连接
- **遗忘策略**：低频记忆移入"地下密室"封存而非删除，可通过特殊路径调取，永不真正丢失

**宫殿进化机制**：
- **生长**：新领域触发新殿自建，频繁访问的房间自动扩容
- **重组**：定期评估宫殿结构合理性，合并冗余房间、拆分过载房间
- **强化**：高频路径形成"主干道"，检索速度持续优化
- **代谢**：地下密室中的记忆若长期未被调取，降级为概要索引以节省空间
